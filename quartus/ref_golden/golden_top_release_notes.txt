-- Release notes for IA-860m
-- For BBRev2, please use files designated for BBRev1.
-- As a general rule, if a fileset is not available for your board revision, please use the nearest number that precedes your board revision.

Version 1.3
-----------
Ensured all general I/O has an associated I/O standard constraint.  
Quartus 25.1.1 fails to generate programming files otherwise.
Rev4 revision added to QPF (resolving a mistake from earlier releases).

Version 1.2
-----------
Removed the BMC_GPIO signals as these should not be used in any design.

Version 1.1
-----------
Updated the DRAM preset to reflect IP change (from 24.3 onwards).  Preset created in Quartus 25.1.

Version 1.0
-----------
Production FPGA listed (BBRev4 fileset).
Extra files added to support BBRev4 (production silicon build)
For production silicon, the PLL placement guidance in the .tcl file is different from other BBRev versions as the internal co-ordinates have changed within the I/O banks.

No changes to the filesets for older BB Revisions.

Version 0.4
-----------
Only ES FPGA listed.
Extra files added to support BBRev3
 - Some BMC interface pin changes
Notes for PLL placement guidance in the .tcl files 
 - This is due to issues that Bittware experienced when the PLL for USER_CLK0 was autoplaced a neighbouring I/O Bank.
HPS DRAM Preset changed
 - Changed to support Quartus 24.2.  Please use presets from prior golden_top release for older Quartus tools.
Some configuration settings changed 
 - ERROR_CHECK_FREQUENCY_DIVISOR & MINIMUM_SEU_INTERVAL
 - This resolves an issue were SDM was not properly reporting some voltages (and temperature sensor 7).

Version 0.3
-----------
Only ES FPGA listed.  
Extra files added to support BB Rev1.
 - Two FPGA devices supported in ia860m_golden_top_bbrev1_es.tcl.
 - Designers need to comment in the device that matches their product configuration.
HBM Fabric Clocks uncommented from constraints as we have a better
understanding on how to connect up those clocks to the fabric.
 - Must use IOPLL before connecting into the fabric.
 - Can be used for general design clocks or to generate clocks to drive logic to interface with NoC initiators to HBM.
BMC_IF_PRESENT_L added as an assignment in the .v and .vhd files.  This assignment should be commented out when the Bittware BMC IP is integrated into the design.
 - This signal must be driven HIGH when Bittware's BMC FPGA IP is not integrated in the design.
vsr_mode setting added to the receivers on the QSFPDDs (VSR_MODE_HIGH_LOSS).  This comes as a recommendation from Altera for NRZ linerates >23Gbps and all PAM4 linerates.

-------------------------------------------------

Version 0.2
-----------
Initial release.
Only ES FPGA listed.
Only BB Rev0 supported.
HBM Fabric Clocks commented out from constraints due to issues building them into designs.
